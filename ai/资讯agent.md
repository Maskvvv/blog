# 用三 Agent 架构打造每日投研机会与风险分析系统

在做股票、行业板块、期货和宏观市场跟踪时，一个常见问题是：信息太多、来源太杂、研报滞后、观点互相冲突，而且很容易被旧闻或单一机构观点误导。

如果我们希望每天早上通过 AI 自动阅读顶级投行、券商研报、企业公告、宏观数据和期货市场信息，并输出一份“今日有哪些机会和风险”的报告，那么最好不要只用一个 Agent 完成所有任务。

更稳妥的设计是将系统拆成三个子 Agent：

```text
资料搜集 Agent
    ↓
研究分析 Agent
    ↓
风控审查 Agent
    ↓
最终投研报告
```

这篇文章整理一个可落地的三 Agent 投研系统设计，并给出基于 Python + OpenAI SDK 的实现思路。

------

## 一、为什么不要只用一个 Agent？

很多人一开始会写一个很长的 System Prompt，让一个 AI 同时完成：

- 搜索最新研报；
- 判断哪些信息重要；
- 分析股票和行业机会；
- 分析期货和商品方向；
- 输出最终报告；
- 自我检查风险。

这种方式看似简单，但有几个明显问题：

1. **容易混淆事实和观点**
   AI 可能把机构观点当成事实，把推理包装成确定性结论。
2. **容易引用旧闻**
   投研信息对时效性要求很高，昨天的结论今天可能已经被市场定价。
3. **容易过度乐观**
   如果模型在资料中看到多个看多观点，可能忽略估值、拥挤交易和反方证据。
4. **缺少独立风控审查**
   自己写报告再自己审查，效果通常不如让一个专门的风控 Agent 来挑刺。

因此，更推荐使用多 Agent 工作流。

------

## 二、整体架构设计

推荐的工作流如下：

```text
定时任务触发
    ↓
资料搜集 Agent
    ↓
结构化信息池
    ↓
研究分析 Agent
    ↓
初版机会与风险报告
    ↓
风控审查 Agent
    ↓
最终报告输出
```

每天早上定时任务运行一次，覆盖过去 24-48 小时内的最新信息。

核心目标不是“推荐买什么”，而是回答：

- 今天哪些资产有机会？
- 哪些行业板块值得观察？
- 哪些期货品种风险上升？
- 哪些机会已经被市场定价？
- 哪些机构观点存在分歧？
- 哪些方向应该降低置信度？
- 哪些结论证据不足，不应该发布？

------

## 三、三个 Agent 的职责划分

## 1. 资料搜集 Agent

资料搜集 Agent 只负责找资料、筛选资料、提取事实，不负责做投资判断。

它的核心任务是：

```text
1. 搜索过去 24-48 小时内的最新投行研报、券商研报、宏观数据、企业公告、期货评论。
2. 优先检查每篇文章或报告的发布时间 Timestamp。
3. 剔除旧闻、重复信息、低质量来源。
4. 提取核心事实、机构观点、数据变化、目标价变化、评级变化。
5. 输出结构化 JSON，不做交易建议。
```

高优先级来源可以包括：

- Goldman Sachs；
- Morgan Stanley；
- J.P. Morgan；
- Bank of America Securities；
- Citi；
- UBS；
- Barclays；
- Deutsche Bank；
- Evercore；
- Lazard；
- 中金公司；
- 中信证券；
- 中信建投；
- 华泰证券；
- 国泰海通证券；
- 招商证券；
- 交易所、监管机构、央行、上市公司公告等。

资料搜集 Agent 最重要的一条规则是：**拒绝旧闻**。

对于每日晨报场景，默认只采用过去 24-48 小时内的信息。超过 48 小时的信息，除非是非常重要的长期深度报告，否则不得作为核心依据。

------

## 2. 研究分析 Agent

研究分析 Agent 接收资料搜集 Agent 输出的结构化 JSON，然后分析机会和风险。

它需要覆盖：

- 宏观环境；
- A股、港股、美股；
- 行业板块；
- 个股观察；
- 商品期货；
- 股指期货；
- 国债期货；
- 汇率和利率；
- 机构观点分歧；
- 未来催化剂。

它不能凭空编造信息，只能基于资料搜集 Agent 提供的内容分析。

它需要区分：

```text
事实：
已经发生的数据、公告、价格变化、政策变化。

机构观点：
投行、券商、资管机构对资产、行业、公司的判断。

模型推理：
基于事实和机构观点得出的可能影响。
```

同时，每个机会都必须配套风险，每个风险都需要触发条件或失效条件。

例如：

```text
半导体：机会偏强。
核心依据：多家机构上调 AI 算力链盈利预期，订单能见度提升。
主要风险：估值处于偏高区间，若财报无法验证盈利弹性，可能出现回调。
是否已定价：部分定价。
置信度：中。
```

------

## 3. 风控审查 Agent

风控审查 Agent 是整个系统里最重要的一环。

它的定位不是找机会，而是专门挑刺：

```text
1. 检查初版报告有没有证据不足。
2. 检查是否用了旧闻。
3. 检查是否忽略风险。
4. 检查是否把机构观点说成事实。
5. 检查是否存在过度确定性表达。
6. 检查是否存在追高、拥挤交易、估值过高风险。
7. 要求降低置信度或删除结论。
```

它需要重点检查：

- 信息是否来自过去 24-48 小时；
- 是否缺少时间戳；
- 是否过度依赖单一来源；
- 是否把媒体转述当作原始报告；
- 是否把机构观点当作事实；
- 是否存在“必涨”“确定性机会”“可以买”等不当表达；
- 高置信度是否真的有至少 3 个高质量来源支持；
- 单一来源结论是否被错误标成高置信度。

风控 Agent 的原则是：

```text
宁可保守，不可过度乐观。
宁可降低置信度，不可夸大确定性。
没有交叉验证的结论必须降级。
没有时间戳的资料必须降权。
风险收益比不清晰，不能进入高优先级观察清单。
```

## 五、基于 Python 的最小可用实现

下面是一份简化版实现思路，基于你原来的代码结构改造而来。

原始逻辑是：

```text
一个 System Prompt
    ↓
一次模型调用
    ↓
输出结果
```

改造后变成：

```text
资料搜集 Agent
    ↓
研究分析 Agent
    ↓
风控审查 Agent
    ↓
最终结果写入 item["json"]["data"]
```

示例代码如下：

```python
from openai import OpenAI
import os
from datetime import datetime, timedelta

MODEL_NAME = "[G渠道][1额度/次]gemini-2.5-flash-search"

client = OpenAI(
    api_key=os.getenv("OPENAI_API_KEY"),
    base_url="https://vip.xiaweiliang.cn/v1"
)

TODAY = datetime.now().strftime("%Y-%m-%d")
TWO_DAYS_AGO = (datetime.now() - timedelta(days=2)).strftime("%Y-%m-%d")
```

注意，不建议把 API Key 直接写死在代码里。应该使用环境变量：

```python
api_key=os.getenv("OPENAI_API_KEY")
```

如果之前已经把 Key 暴露在聊天记录、代码仓库或自动化平台日志中，建议立刻去后台重置。

## 九、通用调用函数

```python
def call_agent(system_prompt, user_prompt):
    response = client.chat.completions.create(
        model=MODEL_NAME,
        messages=[
            {
                "role": "system",
                "content": system_prompt
            },
            {
                "role": "user",
                "content": user_prompt
            }
        ],
        stream=False
    )

    return response.choices[0].message.content
```

8. 股指期货
9. 国债期货
10. 汇率与利率
11. 宏观政策和地缘政治

请重点回答：
- 当前有哪些机会？
- 当前有哪些风险？
- 哪些板块或资产值得进入观察池？
- 哪些方向已经被市场充分定价？
- 哪些方向需要回避？
""")

    collector_user_prompt = f"""
今天是 {TODAY}。

请运行资料搜集任务。

要求：
1. 只采集过去 24-48 小时内的最新信息。
2. 必须检查 Timestamp。
3. 拒绝旧闻。
4. 优先使用顶级投行、券商、官方数据、交易所公告、上市公司公告。
5. 输出严格 JSON。

用户关注范围：
{user_focus}
"""

    collector_output = call_agent(
        system_prompt=COLLECTOR_SYSTEM_PROMPT,
        user_prompt=collector_user_prompt
    )

    analyst_user_prompt = f"""
以下是资料搜集 Agent 输出的结构化资料。

请基于这些资料生成《每日投研机会与风险报告：初版》。

注意：
1. 不得使用资料之外的信息。
2. 资料不足时必须说明。
3. 必须区分事实、机构观点和你的推理。
4. 不得给直接买卖建议。
5. 每个机会都必须有风险、触发条件和失效条件。

资料如下：

{collector_output}
"""

    analyst_output = call_agent(
        system_prompt=ANALYST_SYSTEM_PROMPT,
        user_prompt=analyst_user_prompt
    )

    risk_user_prompt = f"""
请对研究分析 Agent 的初版报告进行风控审查，并输出最终修正版报告。

你需要同时参考：

{collector_output}

二、研究分析 Agent 的初版报告：
{analyst_output}

请重点检查：
1. 是否使用旧闻；
2. 是否缺少时间戳；
3. 是否过度依赖单一来源；
4. 是否把机构观点当事实；
5. 是否存在过度乐观；
6. 是否忽略风险；
7. 是否置信度过高；
8. 是否存在直接买卖建议；
9. 是否需要删除、降级或补充风险。
"""

    final_output = call_agent(
        system_prompt=RISK_SYSTEM_PROMPT,
        user_prompt=risk_user_prompt
    )

    item["json"]["data"] = final_output

    item["json"]["collector_output"] = collector_output
    item["json"]["analyst_output"] = analyst_output
    item["json"]["risk_reviewed_output"] = final_output
    item["json"]["run_date"] = TODAY

return _items

## 十二、推送到飞书时的注意事项

如果最终报告要通过 Webhook 推送到飞书，需要注意内容长度和格式限制。

之前遇到的报错类似：

```json
{
  "code": 19002,
  "msg": "params error, unknown content value"
}
```

以及：

```json
{
  "code": 11246,
  "msg": "card table number over limit"
}
```

这说明报告内容可能存在：

- 表格太多；
- 卡片结构不符合飞书 Post 格式；
- content 字段格式错误；
- Markdown 没有正确转换为飞书消息 JSON；
- 输出太长，超过飞书卡片限制。

因此建议在风控 Agent 或最终报告 Agent 里加上限制：

```text
最终报告控制在 3000 字以内。
最多使用 1 个表格。
其余内容使用 bullet points。
避免复杂嵌套表格。
优先输出最重要的机会、风险和观察清单。
```

如果要推送到飞书，报告结构可以简化成：

```text
# 每日投研机会与风险报告

## 今日核心结论
- ...

## 机会观察
- ...

## 风险提示
- ...

## 重点催化剂
- ...

## 最终观察清单
- ...

## 免责声明
...
```

这样更适合移动端阅读，也更不容易触发飞书 Webhook 限制。

## 十三、这个系统最重要的设计原则

这个每日投研 Agent 的重点不是“让 AI 找牛股”，而是建立一个稳定的信息处理和风控流程。

最重要的原则有七条：

```text
1. 防止旧闻
2. 防止幻觉
3. 防止单一来源误导
4. 防止研报滞后
5. 防止高位追题材
6. 防止把机构观点当事实
7. 防止忽略反方证据
```

三个 Agent 的重要性排序应该是：

```text
风控审查 Agent > 资料搜集 Agent > 研究分析 Agent
```

很多人会把分析 Agent 写得很强，但忽略风控 Agent。对于投研场景，这是很危险的。

因为市场分析最大的风险往往不是“没有观点”，而是“过度自信地相信一个看起来合理的观点”。

## 十四、总结

一个更可靠的每日投研 Agent，不应该只是一个很长的 Prompt，而应该是一个清晰分工的工作流：

```text
资料搜集 Agent：
负责新鲜度、来源质量、结构化事实。

研究分析 Agent：
负责机会、风险、分歧、催化剂和观察清单。

风控审查 Agent：
负责反驳、降置信度、删除证据不足结论、补充风险。
```

最终输出的不是直接买卖建议，而是一份可以辅助决策的《每日投研机会与风险报告》。

它应该告诉你：

- 哪些方向值得观察；
- 机会逻辑是什么；
- 风险在哪里；
- 什么条件下观点成立；
- 什么条件下观点失效；
- 哪些结论证据不足；
- 哪些机会可能已经被市场定价。

对于每天早上自动运行的投研场景，三 Agent 架构已经足够实用。再多的 Agent 反而可能增加成本、延迟和结果分散度。

最推荐的落地版本就是：

```text
Market Research Collector
    ↓
Investment Analysis Agent
    ↓
Risk Committee Agent
    ↓
每日投研机会与风险报告
```

最后，必须始终保留一句话：

```text
以上内容仅为基于公开资料和机构观点的研究分析，不构成任何投资建议。市场有风险，交易需独立决策。
```
